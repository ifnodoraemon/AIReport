# 模型发布追踪

最后更新：2026-03-25

参考文档：`/home/ifnodoraemon/myreport/AI三巨头博客追踪.md`

跟踪范围：近期高信号 AI 模型发布，优先关注 `OpenAI`、`Anthropic`、`Google`，以及和 `coding`、`agent`、`tool use`、`长上下文`、`多模态`、`成本分层` 相关的能力变化

## 目的

这份文件作为长期维护的模型发布记录，用于：

- 统一记录重要模型发布，而不是把信息散落在博客文档里
- 区分 `frontier model`、`高吞吐小模型`、`coding model`、`agent model`
- 追踪模型能力、价格定位、工具能力和适用场景的变化
- 为后续选型、评测和成本判断提供对照基线

## 当前判断

当前最值得关注的高信号主题：

1. 大厂模型发布已经从“更强通用模型”转向“`frontier + 低成本 + agent-friendly`”分层。
2. `coding`、`tool use`、`computer use`、`knowledge work` 正在成为旗舰模型默认卖点。
3. 真正影响工程落地的，不只是 benchmark，而是 `价格 / 吞吐 / 上下文 / 工具调用 / 运行环境` 的组合。

## 当前优先级

### P0

- 跟踪 `frontier model` 的工具能力和长任务能力
- 跟踪 `低成本小模型` 的吞吐、价格和替代性
- 建立统一对照维度：`context`、`tool use`、`coding`、`multimodal`、`price`

### P1

- 补齐更多厂商和开源模型，但只保留高信号发布
- 把博客信号映射到真实模型选型和评测项

## 建议记录字段

后续新增模型时，建议至少记录：

- 模型名
- 公司
- 日期
- 定位
- 核心能力变化
- 工具 / agent 能力
- 成本 / 吞吐信号
- 适合场景
- 不适合场景
- 官方来源
