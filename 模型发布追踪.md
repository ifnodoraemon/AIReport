## 当前判断

当前最值得关注的高信号主题：

1. 大厂模型发布已经从“更强通用模型”转向“`frontier + 低成本 + agent-friendly + domain-specialized`”分层。
2. `coding`、`tool use`、`computer use`、`knowledge work`、`scientific workflow` 正在成为旗舰模型默认卖点。
3. 真正影响工程落地的，不只是 benchmark，而是 `价格 / 吞吐 / 上下文 / 工具调用 / 运行环境 / 访问门控 / 安全约束` 的组合。

## 2026-04-06 当周补充

### 新增模型 / 补录

- 模型：`Claude Opus 4.6`
  公司：`Anthropic`
  定位：`frontier / coding / long-horizon agent`
  核心变化：强调更强 `coding`、更稳定的大代码库操作、`1M context window beta`，并在 `Terminal-Bench 2.0`、`Humanity's Last Exam` 上继续把最强卖点锚定为 agentic work。
  为什么重要：这是 Anthropic 在 `Opus` 线上把“深推理”进一步和“长时 agent 执行”绑定。
  建议动作：继续把它作为 `最强 agent/coding` 档位的对照基线。
  来源：https://www.anthropic.com/news/claude-opus-4-6

- 模型：`Claude Sonnet 4.6`
  公司：`Anthropic`
  定位：`cost-effective frontier / agent-default`
  核心变化：在更低成本档位上给出更强的 `coding`、`computer use`、`long-context reasoning`、`agent planning`，并公开强调 `1M context beta`。
  为什么重要：说明 Anthropic 正把强 agent 能力往更便宜、默认的主力模型下沉。
  建议动作：模型选型上不要再简单用 `Sonnet=便宜, Opus=强` 这种旧二分法。
  来源：https://www.anthropic.com/news/claude-sonnet-4-6

- 模型：`Gemma 4`
  公司：`Google`
  定位：`open model / advanced reasoning / agentic workflows`
  核心变化：直接把 `function calling`、`structured JSON output`、`system instructions` 和 `agentic workflows` 作为开放模型能力来卖。
  为什么重要：这是一条明确的“开放模型也要 agent-friendly”信号。
  建议动作：把 `Gemma 4` 加入开源 / 开放模型对照表，不要只用它做轻量本地模型参考。
  来源：https://blog.google/innovation-and-ai/technology/developers-tools/gemma-4/

- 模型：`Gemini 3.1 Flash Live`
  公司：`Google`
  定位：`real-time audio / live interaction`
  核心变化：更低延迟、更自然的语音对话，目标是让实时语音交互成为默认表面。
  为什么重要：语音模型正在从“语音转文本配件”升级成独立模型线，这会影响 agent 入口设计。
  建议动作：若后续关注客服、助手或会议场景，应把 `live audio` 作为单独评测维度。
  来源：https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-1-flash-live/

### 状态变化

- 模型分层：
  之前判断：大厂正在形成 `frontier + 低成本 + agent-friendly` 三层结构。
  当前判断：这套结构进一步清晰，但 `agent-friendly` 已不只属于旗舰模型，也在往 `Sonnet` 和 `Gemma` 这类更可部署档位下沉。
  变化原因：Anthropic 与 Google 都开始在更低成本或更开放的模型线上显式卖 `agentic workflows`。

- OpenAI 状态：
  之前判断：`GPT-5.4` 是当前主线。
  当前判断：截至 `2026-04-06`，OpenAI 公开主发布页仍以 `GPT-5.4` 为最新旗舰主线，本周没有同级别独立新模型发布页需要追加。
  变化原因：最新公开增量更多出现在 `spec / monitoring / product behavior`，而不是新模型代际。

## 2026-04-11 当周补充

### 新增模型 / 补录

- 模型：`Mythos preview`
  公司：`Anthropic`
  定位：`restricted preview / frontier security-oriented model`
  核心变化：截至 `2026-04-11`，Anthropic 官方站点已把 `Mythos preview` 列入模型入口；与 `2026-04-07` 的 `Project Glasswing` 同期出现，说明这是一条受限预览、非公开普发的最新模型线。
  为什么重要：这意味着 Anthropic 最新最强模型信号已经不止停留在 `Opus 4.6 / Sonnet 4.6`，而是进入了“高风险能力先受控部署”的阶段。
  建议动作：在模型追踪里单独标记 `restricted preview` 这一类，不要把它和公开 API / Claude.ai 普发模型混在一起。
  来源：https://www.anthropic.com/news ; https://www.anthropic.com/news?prog_id=13

- 模型：`GPT-5.4 mini`
  公司：`OpenAI`
  定位：`small model / coding / subagents`
  核心变化：OpenAI 公开把它写成更快、更便宜、适合 `coding assistants`、`computer use`、`subagents` 的执行层模型，并给出 `400k context`、`skills`、`computer use`、`web search` 等完整 agent 能力接口。
  为什么重要：这让 OpenAI 的模型分层第一次非常明确地服务于 `agent system design`，而不是只服务于聊天产品分层。
  建议动作：把 `GPT-5.4 mini` 作为高频执行、并行子任务和低成本 coding agent 的新基线。
  来源：https://openai.com/index/introducing-gpt-5-4-mini-and-nano/

- 模型：`GPT-5.4 nano`
  公司：`OpenAI`
  定位：`ultra-low-cost / extraction / ranking / simple coding subagents`
  核心变化：OpenAI 把它定位为最小、最便宜的 `GPT-5.4` 变体，适合分类、抽取、排序和较轻量的 coding subagents。
  为什么重要：这说明 OpenAI 已明确在 agent 体系内预留“极低成本执行层”。
  建议动作：后续在需要大规模并发或流水线拆分时，把 `nano` 纳入 cost baseline。
  来源：https://openai.com/index/introducing-gpt-5-4-mini-and-nano/

- 模型：`Gemini 3.1 Pro`
  公司：`Google`
  定位：`core reasoning / complex tasks / developer default`
  核心变化：Google 把 `3.1 Pro` 作为更普遍的复杂任务底座，并同步接入 `Gemini API`、`Vertex AI`、`Gemini app`、`NotebookLM`、`Gemini CLI` 和 `Antigravity`。
  为什么重要：这让 Google 的 developer stack 不再只有 `Deep Think` 这种高端推理档位，而有了更清楚的中枢模型。
  建议动作：在 Google 侧评测中，把 `3.1 Pro` 和 `Deep Think / Flash-Lite / Flash Live` 一起看作分层组合，而不是只盯一个旗舰点位。
  来源：https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-1-pro/

### 状态变化

- OpenAI 模型分层：
  之前判断：`GPT-5.4` 是主线，其他增量更像产品能力补充。
  当前判断：OpenAI 已把 `GPT-5.4 -> mini -> nano` 明确做成 `planner -> executor -> cheap executor` 的可组合体系。
  变化原因：`mini / nano` 发布页直接把 `subagents`、`computer use`、`skills` 写进定位。

- Google 模型分层：
  之前判断：Google 的主线是 `Deep Think + Flash-Lite + Gemma 4`。
  当前判断：还要把 `Gemini 3.1 Pro` 视作开发者默认底座，否则会高估 Google 对专门档位的依赖。
  变化原因：Google 已把 `3.1 Pro` 分发到开发者、企业和消费者多个入口。

- Anthropic 最新模型状态：
  之前判断：最新公开模型发布是 `Claude Sonnet 4.6`。
  当前判断：如果只看“公开产品发布页”，仍然是 `Sonnet 4.6`；但如果看 `2026-04-11` 官方站点当前模型入口，最新更强的受限模型信号已经变成 `Mythos preview`。
  变化原因：Anthropic 已在官方站点的模型入口显示 `Mythos preview`，但没有用常规公开产品发布页大规模铺开。

## 2026-04-17 当周补充

### 新增模型 / 补录

- 模型：`Gemini 3.1 Flash TTS`
  公司：`Google`
  定位：`audio generation / expressive speech / developer distribution`
  核心变化：Google 推出新的文本转语音模型，强调 `granular audio tags`、`70+ languages`、更强可控性与更自然的表达，并同步进入 `Gemini API`、`Vertex AI`、`Google Vids`。
  为什么重要：这说明 Google 已把 `speech generation` 做成单独模型层，而不是从聊天模型附带导出语音。
  建议动作：如果后续涉及客服、配音、语音助手，应把 `TTS controllability` 和 `watermarking` 单列成评测项。
  来源：https://blog.google/innovation-and-ai/models-and-research/gemini-models/gemini-3-1-flash-tts/

- 模型：`Gemini Robotics-ER 1.6`
  公司：`Google`
  定位：`embodied reasoning / physical agent model`
  核心变化：这条模型线明确支持 `spatial reasoning`、`success detection`、`instrument reading`，并把 `agentic vision + code execution + external tools` 组合到 physical agent 工作流中。
  为什么重要：这不是普通多模态升级，而是把 agent 模型从数字界面延伸到实体执行环境。
  建议动作：在模型分层里单独保留 `robotics / embodied reasoning`，不要全部并到通用 Gemini 线。
  来源：https://deepmind.google/blog/gemini-robotics-er-1-6/

### 状态变化

- Google 模型分层：
  之前判断：Google 在 `Deep Think / Pro / Flash-Lite / Flash Live / Gemma 4` 上已经形成较完整组合。
  当前判断：还应再显式加入 `Flash TTS` 与 `Robotics-ER 1.6` 两条专门模型线，Google 的分层已扩展到 `语音生成 + embodied reasoning`。
  变化原因：`2026-04-14` 和 `2026-04-15` 两篇官方发布页都不是轻量产品更新，而是独立模型能力层补位。

- OpenAI / Anthropic 本周模型状态：
  之前判断：OpenAI 最近主增量在 runtime 和 Codex，Anthropic 最近主增量在 `Mythos preview` 与 managed agents。
  当前判断：截至 `2026-04-17`，这两个判断保持不变；本周没有看到同级别的公开新模型发布页需要改写主线。
  变化原因：两家新增主要集中在平台与工作流，而不是新模型代际。

## 2026-04-28 当周补充

### 新增模型 / 补录

- 模型：`GPT-5.5`
  公司：`OpenAI`
  定位：`frontier / agentic coding / knowledge work / scientific research`
  核心变化：GPT-5.5 已替代 `GPT-5.4` 成为 OpenAI 最新公开主线，强调更强 `agentic coding`、`computer use`、`knowledge work`、`scientific research`，并在 Codex 中提供 `400K context`，API 侧更新后提供 `1M context`。

  建议动作：把 `GPT-5.5` 设为 OpenAI 侧最新 frontier baseline，同时记录它的 `cyber / bio` 高风险能力门控。
  来源：https://openai.com/index/introducing-gpt-5-5/

- 模型：`Claude Opus 4.7`
  公司：`Anthropic`
  定位：`frontier / coding / long-running agents / vision`
  核心变化：Opus 4.7 已成为 Anthropic 最新公开可用 Opus 模型，重点提升复杂软件工程、长时任务、指令遵循、视觉分辨率、文件系统 memory，并加入 `xhigh effort`、`task budgets` 等运行控制。
  为什么重要：Anthropic 的前沿模型能力进一步和 agent harness、预算控制、review、auto mode 绑定。
  建议动作：把 `Opus 4.7` 设为 Claude 侧公开可用 frontier baseline；`Mythos preview` 仍单独标记为受限预览。
  来源：https://www.anthropic.com/news/claude-opus-4-7

- 模型：`GPT-Rosalind`
  公司：`OpenAI`
  定位：`domain-specialized / life sciences / trusted access`
  核心变化：OpenAI 发布面向生命科学研究的专用模型系列预览，覆盖生物、药物发现、转化医学、科学工具调用，并通过 trusted access 限定可用范围。
  为什么重要：这说明 OpenAI 不只在通用 frontier 上迭代，也开始把高风险高价值领域做成专用模型线。
  建议动作：后续模型分层中新增 `domain-specialized frontier model`，不要把它和普通工具插件混在一起。
  来源：https://openai.com/index/introducing-gpt-rosalind/

### 状态变化

- OpenAI 最新模型状态：
  之前判断：`GPT-5.4` 是当前主线，`mini / nano` 是执行层补充。
  当前判断：`GPT-5.5` 已成为最新公开 frontier 主线，`GPT-5.4` 降为上一代基线；同时 `GPT-Rosalind` 显示 OpenAI 正在开出高门槛领域模型线。
  变化原因：OpenAI 在 `2026-04-23` 发布 GPT-5.5，并在 `2026-04-16` 发布 GPT-Rosalind 研究预览。

- Anthropic 最新模型状态：
  之前判断：`Mythos preview` 是更强受限模型信号，公开可用主线仍偏 `Opus / Sonnet 4.6`。
  当前判断：`Claude Opus 4.7` 已成为公开可用最新 Opus；`Mythos preview` 仍代表更高风险能力的 gated deployment 路线。
  变化原因：Opus 4.7 已在 Claude 产品、API、Bedrock、Vertex AI、Microsoft Foundry 可用。

- Google 最新模型状态：
  之前判断：Google 本月新增重点是 `Flash TTS` 与 `Robotics-ER 1.6`。
  当前判断：截至 `2026-04-28`，本周没有看到同级别独立新 Gemini 模型发布；Google 主要增量转向 `Gemini Enterprise Agent Platform` 与训练基础设施。
  变化原因：Cloud Next 26 的重点在平台、runtime、governance、TPU 和 data cloud，而不是单个模型代际。

## 2026-05-07 当周补充

### 新增模型 / 补录

- 模型：`GPT-5.5 Instant`
  公司：`OpenAI`
  定位：`default ChatGPT model / low-latency general model / personalized assistant`
  核心变化：替代 `GPT-5.3 Instant` 成为 ChatGPT 默认模型，并在 API 中作为 `chat-latest` 暴露；重点改善事实性、图像理解、STEM、web search 决策、简洁性和个性化上下文使用。
  为什么重要：这是 OpenAI 最高频入口模型的升级，不是 frontier thinking 模型，但会影响最多真实用户工作流。
  建议动作：把 `Instant` 档位单独建为 `default / high-volume / personalization` 基线，不和 `GPT-5.5 Thinking` 混写。
  来源日期：`2026-05-05`
  来源：https://openai.com/index/gpt-5-5-instant/ ; https://openai.com/index/gpt-5-5-instant-system-card/

- 模型：`Gemma 4 MTP drafters`
  公司：`Google`
  定位：`open model inference acceleration / speculative decoding`
  核心变化：Google 为 Gemma 4 family 发布 Multi-Token Prediction drafters，用专门 speculative decoding 架构提升推理速度，同时强调不损失输出质量和 reasoning logic。
  为什么重要：这不是新一代模型，但会改变开放模型的实际可用性，尤其是本地、端侧和高并发开发者场景。
  建议动作：在开放模型追踪中把 `base model capability` 和 `serving/inference add-on` 分开记录。
  来源日期：`2026-05-05`
  来源：https://blog.google/innovation-and-ai/technology/developers-tools/multi-token-prediction-gemma-4/

### 状态变化

- OpenAI 最新模型状态：
  之前判断：`GPT-5.5` 是 OpenAI 最新公开 frontier 主线。
  当前判断：判断保持不变，但 `GPT-5.5 Instant` 成为新的默认高频模型入口，应作为独立 baseline；系统卡显示默认模型也需要高能力类别安全分级。

### 横向比较

- 对比项：`frontier model` vs `default model` vs `inference acceleration`
  结论：OpenAI 本周补的是默认入口模型，Anthropic 补的是可用性和行业模板，Google 补的是开放模型推理效率。
