## 比较目的

这页不是比“谁更强”，而是比两家在 `agent stack` 上各自更像什么。

## 当前结论

### 1. OpenAI 更像 `agent platform company`

核心特征：

- `GPT-5.4`
- `Responses API`
- `computer environment`
- `Model Spec`
- `monitoring`

更强的信号是：

- `runtime`
- `execution environment`
- `spec + eval + monitoring`

### 2. Anthropic 更像 `coding agent + MCP company`

核心特征：

- `Claude Opus 4.6 / Sonnet 4.6`
- `Claude Code`
- `MCP`
- `long-running harness`
- `agent eval`

更强的信号是：

- `coding`
- `MCP ecosystem`
- `harness discipline`
- `eval discipline`

## 主要差异

### OpenAI

- 更强调平台层整合
- 更强调 runtime、spec、治理层
- 更像在构建“默认 agent 操作系统”

### Anthropic

- 更强调 coding 和长流程工作方式
- 更强调 MCP 和 handoff 工件
- 更像在构建“高纪律性的 coding agent 栈”

## 对我们的影响

如果我们更关心：

- `runtime / governance / execution environment`
  更该重点学 `OpenAI`

- `coding workflow / MCP / long-running harness / eval`
  更该重点学 `Anthropic`

## 当前建议

- 不要把它们当同一路线的两个品牌
- 更应该把它们视作两种不同的 agent stack 取向
